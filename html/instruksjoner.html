Instruksjonsøvelser for UFW

Disse øvelsene kan utføres på en lab-maskin som støtter og kjører UFW. Målet er å forstå grunnleggende funksjoner i UFW, lage regler og observere hvordan de fungerer.

Oppgave 1: Aktiver og sjekk UFW-status

1. Logg inn på lab-maskinen.

2. Sjekk gjeldende status for UFW:

sudo ufw status

3. Aktiver UFW:

sudo ufw enable

4. Bekreft at UFW er aktiv ved å sjekke status på nytt.

Oppsummering: Du har aktivert UFW og lært hvordan du sjekker om brannmuren er aktiv. Dette gir grunnleggende beskyttelse for systemet ditt.

Oppgave 2: Tillat lokal tilgang

1. Legg til en regel som kun tillater trafikk på en valgt port (for eksempel 8080) fra localhost:

sudo ufw allow from 127.0.0.1 to any port 8080

2. Verifiser regelen ved å liste opp UFW-regler:

sudo ufw status

3. Test forbindelsen lokalt.

Oppsummering: Du har konfigurert UFW til å tillate trafikk kun fra localhost. Dette sikrer at ingen eksterne tilkoblinger er mulig for denne porten.

Oppgave 3: Tillat tilgang fra et spesifikt LAN

1. Tillat tilkoblinger til en valgt port (for eksempel 22 for SSH) kun fra ditt LAN (for eksempel 10.12.X.0/24):

sudo ufw allow from 10.12.X.0/24 to any port 22

2. Verifiser at kun enheter fra ditt LAN har tilgang.

Oppsummering: Du har lært å begrense tilgang til en port basert på nettverkssegmenter, noe som forbedrer sikkerheten ved å tillate kun lokale enheter.

Oppgave 4: Blokker trafikk fra en IP

1. Identifiser en test-IP-adresse (for eksempel 10.12.X.100) som skal blokkeres.

2. Blokker all trafikk fra denne IP-en:

sudo ufw deny from 10.12.X.100

3. Bekreft blokkeringen ved å teste forbindelsen fra den blokkerte IP-en.

Oppsummering: Du har blokkert trafikk fra en spesifikk IP. Dette er nyttig for å hindre uønsket eller mistenkelig aktivitet på serveren.

Oppgave 5: Aktiver logging

1. Slå på UFW-logging:

sudo ufw logging on

2. Vis loggen for å se tillatte og avviste forbindelser:

sudo less /var/log/ufw.log

3. Observer hvordan reglene dine reflekteres i loggen.

Oppsummering: Ved å aktivere logging kan du overvåke hvilke tilkoblinger som er tillatt eller avvist, noe som gir innsikt i trafikken som går gjennom systemet.

Oppgave 6: Test ekstern tilgang

1. Tillat tilgang til en valgt port (for eksempel 80 for HTTP) fra en ekstern enhet eller VPN:

sudo ufw allow from <ekstern-IP-område> to any port 80

2. Verifiser at den eksterne enheten får tilgang.

Oppsummering: Du har konfigurert UFW til å tillate ekstern tilgang, noe som er nyttig for å sikre tjenester for eksterne brukere.

Oppgave 7: Tilbakestill regler

1. Tilbakestill UFW til standardinnstillingene:

sudo ufw reset

2. Konfigurer UFW med minimumsregler som tillater:

  • Lokal tilgang til nødvendige tjenester.
  • Tilgang fra spesifikke LAN-enheter.
  • Eventuell ekstern tilgang som trengs.

Oppsummering: Ved å tilbakestille UFW og legge til minimumsregler, kan du sikre at brannmuren kun tillater nødvendig trafikk.
